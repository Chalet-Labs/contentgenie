## Design Context

### Users

Individual professionals who use ContentGenie to quickly triage podcast content. Their primary mindset is efficiency — they want a fast overview of what's important, not a deep browsing experience. They're scanning between meetings or during transitions, so every element must earn its screen real estate.

### Brand Personality

**Smart. Efficient. Modern.**

The interface should feel like a sharp tool, not a toy. It respects the user's time and intelligence. No unnecessary ornamentation, no playful gimmicks — just clear, well-organized information that helps them decide what's worth their attention.

### Aesthetic Direction

**Editorial-clean** — information-dense but well-organized, with strong typographic hierarchy and generous whitespace between sections. Content is the decoration.

- **Visual tone:** Restrained, precise, confident. Closer to Linear or Readwise Reader than Spotify or Apple Podcasts.
- **References:** Linear (information density + speed), Readwise Reader (content consumption UX), Superhuman (triage-oriented design).
- **Anti-references:** Overly playful or illustration-heavy interfaces. Cluttered dashboards with competing visual elements. Gratuitous gradients or decorative flourishes.
- **Theme:** Light and dark mode (system-aware). Dark near-black zinc (`#09090b`) as the theme anchor.
- **Typography:** Inter — clean, neutral, highly legible at small sizes. Use weight and size for hierarchy, not decoration.
- **Color strategy:** Neutral chrome (current shadcn zinc palette) with a purposeful brand accent for interactive elements, scores, and status indicators. Color should inform, not decorate.
- **Spacing:** Consistent use of Tailwind's spacing scale. Generous vertical rhythm between sections; tighter spacing within related groups.
- **Border radius:** 0.5rem base (`--radius`) — rounded but not bubbly.

### Design Principles

1. **Clarity over cleverness** — Every element should communicate its purpose instantly. If a user has to think about what something means, it's failed.
2. **Information hierarchy first** — Use size, weight, and spacing to guide the eye. The most important information (scores, key takeaways, episode titles) should dominate; metadata should recede.
3. **Respect the scan** — Users are triaging, not exploring. Design for glanceability: scannable lists, visible scores, concise summaries. Minimize clicks to insight.
4. **Purposeful color** — Use color sparingly and meaningfully. Accent color marks interactive elements and important signals (worth-it scores, new indicators). Never use color purely for decoration.
5. **Quiet confidence** — The interface should feel polished and intentional without drawing attention to itself. No loading spinners where skeletons work. No modals where inline actions suffice. No animations that delay access to information.

### Accessibility

- Target: WCAG 2.1 AA compliance
- Minimum contrast ratio: 4.5:1 for body text, 3:1 for large text and UI components
- All interactive elements must be keyboard accessible with visible focus indicators
- Support `prefers-reduced-motion` — animations should enhance, never block
- Semantic HTML and proper ARIA labels for screen reader support
- Touch targets: minimum 44x44px on mobile
